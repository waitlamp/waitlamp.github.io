拯救者R7000 Arch Linux 安装小记

发表于 2021-08-23

前言

本来发行版是打算试试 Debian 11, 直连模式下安装相当顺利，后来装 NV 闭源驱动 Wayland 炸了，回滚开源驱动哦nouveau无法恢复（虽然 Wayland 目前不够完善，但强迫症属实难受），混合模式重新安装直接无法启动，尝试加载AMD GPU 固件修复仍然无法启动，遂连夜逃离到 Arch

准备

U盘Arch 盒装安装介质
良好的网络
没有困难的工作，只有勇敢的打工人的决心。

此外，建议白天安装，否则笔电的响铃绝对提神醒脑ORZ。

制作启动镜像

使用 rufus 或者 balenaetcher 均可

安装过程（有空再补）

网络安装教程需要注意的是，实际安装时，lsblk命令看到的是类似nvme1n1p1的分区。不要生搬硬套sda1
借用 Debian 安装手册 的一句话 (?为什么要在arch安装小记引用Debian安装手册)

两分钟的思考可能会为您节省几个小时的不必要的恢复工作。

屏蔽 Nvidia 独显

首先，需要一个Linus怒斥英伟达名场面：

这样做有几个原因 + Optimus 实在让人头疼 + NV 的闭源驱动暂时还未支持 Wayland + 开源驱动 nouveau 在 wayland 上也有点小问题（之前在 debian 测试 alacritty 输入卡顿） + 我暂时没有什么炼丹需求，不需要 Cuda

所以这里使用简单粗暴的方法，直接禁用nvidia独显。
使用 acpi_call
原项目太久没有维护，这里使用nix-community维护的版本。
